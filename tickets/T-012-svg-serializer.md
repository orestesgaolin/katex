# T-012 — SVG serializer + embedded fonts

**Milestone:** M4  **Status:** review  **Depends on:** T-009, T-003

## Goal
Serialize the box tree to a self-contained SVG string.

## Scope
- `lib/src/svg/svg_serializer.dart` — walk `BoxNode`:
  - `GlyphNode` → `<text>` with the right `font-family`/`font-size`, positioned by
    baseline; apply italic/skew offsets.
  - `RuleNode` → `<rect>`.
  - `HBox`/`VBox`/`VList`/`SpanNode` → nested `<g transform="translate(...)">`.
  - color from `SpanNode`.
- Coordinate system: convert KaTeX em-based dims to SVG user units at the target font
  size; the root `<svg>` width/height/viewBox derived from root box height+depth+width.
- Self-containment: `lib/src/svg/embedded_fonts.dart` (generated by
  `tool/gen_font_bytes.dart`, T-003) → `@font-face` data-URIs in `<defs><style>`.
- `renderToSvg(tex, options)` in `lib/katex.dart` built on `renderToBox`.

## Acceptance criteria
- `dart analyze` clean.
- `renderToSvg('\\frac{a}{b}')` returns well-formed SVG (parses as XML) with embedded
  `@font-face` and at least one `<text>` and the fraction `<rect>`.
- Output for every gallery expression is non-empty valid XML. `test/svg_test.dart` passes.
- Visual correctness vs KaTeX is validated in T-014.
